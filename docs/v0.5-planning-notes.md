## Design decisions worth remembering

- **OSV is default-on** (osv-scanner/Trivy norm), sends only names+versions, `--offline` for air-gap.
  Network failure degrades to the curated `KNOWN_BAD_VERSIONS` list (never fails the scan).
- **Honest VEX**: `exploitable` only when a co-present credential is `--verify`-confirmed live
  (`cred_verified_co_present`, set post-verify in the CLI) or OSV flags known-exploited. Never `not_affected`.
- **Severity** stays a plain string across the supply-chain finding path (consistent with
  `known_bad_versions` etc.); produced only internally by `cvss.bucket()`/`severity_from_osv()`.
- **CVSS** computed in-house (no `cvss` dep) â€” verified against 4 official v3.1 reference vectors.
- **cyclonedx-python-lib** floor `>=8.0.0` (postdates the `V1_6` schema + `ToolRepository` API);
  verified end-to-end against 11.7.0. Components use an explicit `bom_ref=purl` so VEX affects-refs link.
- Lock-file parsers already capture `integrity_hash` + `resolved_url` â†’ poisoning checks are cheap in v0.6.
